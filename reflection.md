# Reflection

## QR1
The most difficult concept for me was async and await.
At first I did not understand why a function returns Future.
After testing Future.delayed in DartPad I understood that
asynchronous programming allows tasks to run without blocking the program.

## QR2
If we change List<int> to List<double> we would need to change
the list type and function parameter types.
This shows that choosing correct data types during design is important.

## QR3
In a real Flutter app asynchronous operations are used when fetching
data from APIs such as weather or user data.
While waiting the UI can display a loading indicator.

## QR4
Separate methods improve readability and organization.
However one function with an operation parameter could reduce repetition.
Both approaches have advantages.
